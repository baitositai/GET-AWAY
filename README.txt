
2Dの鬼ごっこゲームになります
自身の経路探索の勉強で製作しました
敵の移動に簡単にですが経路探索を使用しています



<勉強メモ>
・ダイクストラの処理は別クラスで独立させて,再利用や保守、拡張性をよくする

・priority_queueを使うのは、ダイクストラ法では「現在最も距離が短いノード」
を毎回取り出す必要があるため、priority_queueを使うことで、コストの小さいノードから探索を進めることが可能

・基本的にはGameSceneに持たせる。ステージ情報(マップ)を持っているので、PathFinderを一元管理しやすい。敵やプレイヤーからGameScene::GetPathFinder()などで取得できると便利やね

SetMap()はマップが変わったときに一度だけ呼び出す

FindPath()はプレイヤーの目的地が変わった時に呼ぶ

<補足>

敵AIがプレイヤーを追いかける仕様の場合,プレイヤーの位置を一定間隔で目標にしてFindPath()を再度呼び出すとよい
経路の再計算が重いなら「1フレームに１体だけ計算」など負荷分散をする
